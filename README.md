## 📌 Про проєкт

**DigiUni Registry** — це Java-застосунок з інтерактивним консольним інтерфейсом (CLI). Програма дозволяє керувати даними університету (факультетами, кафедрами, студентами та викладачами) із збереженням стану у JSON-файли. Включає систему авторизації з рольовою моделлю доступу (Admin, Manager, User).

### ✨ Основні можливості (Features)
* 🔐 **Система авторизації:** Захищений вхід та розмежування прав доступу.
* 🏛️ **Управління структурою:** CRUD-операції для факультетів та кафедр.
* 🧑‍🎓 **Управління персоналом:** Додавання, редагування, видалення та переведення студентів і викладачів.
* 📊 **Аналітика та Звіти:** Алгоритми пошуку та генерації звітів (фільтрація, сортування, групування) за допомогою **Java Stream API**.
* 🛡️ **Надійна валідація:** Перевірка даних за допомогою регулярних виразів та використання **Custom Annotations** + **Reflection API**.
* 💾 **Збереження даних:** Автоматична серіалізація/десеріалізація об'єктів у JSON за допомогою Jackson.
* 🖥️ **Інтерактивний CLI:** Зручне кольорове меню завдяки бібліотеці JLine3.
* 🌐 **Мережева взаємодія:** Базовий функціонал TCP-клієнта-сервера за допомогою java.net.Socket.

## 🚀 Встановлення та запуск

### Вимоги (Prerequisites)
Перед запуском переконайтеся, що на вашому комп'ютері встановлені:
* [Java JDK 17](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) або вище.
* [Apache Maven](https://maven.apache.org/download.cgi).
* Git.

### Інструкція з розгортання
1. **Клонуйте репозиторій:**
   
   git clone [https://github.com/dmytro-software/DigiUniLabWorkVladAndDima.git](https://github.com/dmytro-software/DigiUniLabWorkVladAndDima.git)
   cd DigiUniLabWorkVladAndDima
   

2. **Зберіть проєкт за допомогою Maven:**
   
   mvn clean install
   

3. **Запустіть програму:**
   
   mvn exec:java -Dexec.mainClass="Project.Main"
   
   *(Або запустіть клас Main.java безпосередньо з вашої IDE, наприклад IntelliJ IDEA чи Eclipse).*

## 📖 Як користуватися (Usage)

При першому запуску програми автоматично генеруються системні файли бази даних (у папці data/).

1. **Авторизація:** 
   Для першого входу використовуйте обліковий запис адміністратора (створюється автоматично, якщо база порожня):
   * **Логін:** admin
   * **Пароль:** admin

2. **Навігація:**
   Увійшовши в систему, введіть команду help, щоб побачити повний список доступних команд.

3. **Приклади команд:**
   * ls fac — показати всі факультети.
   * add stu — додати нового студента (запустить покроковий майстер).
   * find stu -p — пошук студента за ПІБ.
   * repo dep -s -c — згенерувати звіт: студенти кафедри згруповані за курсом.

4. **Збереження та вихід:**
   ОБОВ'ЯЗКОВО використовуйте команду exit для коректного збереження всіх змін у файли перед закриттям програми!

## 📂 Структура проєкту
Проєкт побудований за принципами чискої архітектури (Clean Architecture):
* Models/ — класи сутностей предметної області.
* Repository/ — рівень доступу до даних (збереження в JSON).
* service/ — бізнес-логіка програми.
* ui/ — консольні обробники та взаємодія з користувачем.
* validation/ & Exceptions/ — контроль якості даних та обробка помилок.
